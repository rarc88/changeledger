## Request

Endurecer `sl check`: specs huérfanos (sin change que los haya graduado), `updated` obsoleto, y enlaces rotos entre changes y specs.

## Investigation

- `sl graduate` deja dos marcadores: en el `## Log` del change escribe
  `graduado a spec \`<archivo>\``; en el cuerpo del spec escribe
  `Graduado del change <id>`. Son los enlaces change↔spec.
- `checkRepo` hoy solo valida changes (su firma ignora `specs`), aunque `loadRepo`
  sí carga los specs. Hay que pasarlos al validador.
- "updated obsoleto": un spec enlazado a un change cuya última actividad
  (`created` o el último timestamp del Log) es posterior a `spec.updated` quedó
  desfasado — la verdad se movió y no se actualizó la fecha.
- Severidad: enlaces rotos son **errores** (apuntan a algo inexistente);
  huérfano y desfasado son **warnings** (no rompen, pero avisan). Un `updated`
  con formato ISO inválido sí es error.

## Proposal

`checkRepo` recibe también `specs`. Nuevo bloque agregado (solo repo completo):

- Construir enlaces: change→spec (Log `graduado a spec \`X\``) y spec→change
  (cuerpo `Graduado del change <id>`).
- **Enlace roto** (error): un change gradúa a un spec inexistente; un spec
  referencia un change inexistente.
- **Huérfano** (warning): spec sin enlace entrante de ningún change y sin
  backlink válido a un change existente.
- **updated** (error si formato ISO inválido; warning si desfasado respecto a la
  actividad de un change enlazado).

Descartado: depender del reloj para detectar `updated` futuro — el validador es
puro (sin reloj); el desfase se mide contra timestamps presentes en los docs.

## Specification

### CR1 — Enlace roto change↔spec
- **Given** un change cuyo Log gradúa a un spec inexistente
- **When** ejecuto `sl check`
- **Then** es un error
- **And** un spec que referencia un change inexistente también es error

### CR2 — Spec huérfano
- **Given** un spec sin change que lo gradúe ni backlink a un change existente
- **When** ejecuto `sl check`
- **Then** se emite un warning de huérfano (no error)

### CR3 — updated desfasado / inválido
- **Given** un spec con `updated` anterior a la actividad de un change enlazado
- **When** ejecuto `sl check`
- **Then** se emite un warning de desfase
- **And** un `updated` con formato ISO inválido es error
