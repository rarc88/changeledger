## Request
`test/cli.test.mjs` monkey-patches `console.log` and `console.error` in a few
tests to hide expected command output. This works while tests run serially, but
it leaks process-global state into the suite and makes future parallel or
selective execution fragile.

The change is intentionally limited to command output capture. Filesystem setup
inside tests may keep using temporary directories and direct `fs` calls.

## Plan
- [x] Add an optional lightweight logger/output dependency to command functions that currently write directly to `console` and are exercised by `test/cli.test.mjs`. — 2026-06-16T20:50:52Z
- [x] Update `test/cli.test.mjs` to pass mock loggers/output sinks instead of replacing `console.log` or `console.error`. — 2026-06-16T20:50:55Z
- [x] Keep CLI behavior unchanged when no logger/output dependency is provided. — 2026-06-16T20:50:57Z
